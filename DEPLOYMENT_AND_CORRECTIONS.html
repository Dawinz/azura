Backend (Railway)

What Was Done

  • Backend deployed on Railway with PHP 8.2 and MySQL.
  • Clean URLs via router.php (no index.php in paths).
  • API v1 routes under /v1/ for the Flutter app (auth, categories, products, etc.).
  • CORS configured for web (localhost, Vercel) and kept consistent on errors.
  • CSRF disabled for API routes so web/mobile can call the API without tokens.
  • Login and register fixed (form data parsing, CORS, redirects).

Deployment Process

  1. Push to main: git push origin main
  2. Railway builds and deploys (Nixpacks, PHP server with router.php).
  3. No manual deploy step; GitHub → Railway is automatic.

Key Backend Files

FilePurpose

Main API Endpoints (v1)

  • POST /v1/user/register — JSON body
  • POST /v1/auth/login — form-urlencoded
  • GET /v1/create_user — query params (utility)
  • GET /v1/category/list, GET /v1/product/list, GET /v1/product/detail_get?slug=...&user_id=...

Frontend Web (Vercel)

What Was Done

  • Flutter web app deployed on Vercel.
  • GitHub: repo Dawinz/azura-frontend, branch main.
  • Vercel: project azura-frontend; build = Flutter web.
  • Build fix: Flutter PATH in vercel.json so flutter is available in both install and build steps.

Deployment Process

  1. Push to main on Dawinz/azura-frontend.
  2. Vercel runs Install (clone Flutter SDK), then Build: export PATH=".../_flutter/bin:$PATH" && flutter pub get && flutter build web --no-wasm-dry-run, Output: build/web.

Mobile Apps (APK / App Store / Google Play)

What Was Done

  • iOS: App prepared and uploaded to App Store.

Android (APK / Google Play)

  • Build command: flutter build apk --release
  • Output: azura-frontend/build/app/outputs/flutter-apk/app-release.apk
  • Config: android/app/build.gradle.kts (applicationId com.example.shop), AndroidManifest.xml (INTERNET, ACCESS_NETWORK_STATE).
  • Store: App prepared and submitted to Google Play.

iOS (App Store)

  • App prepared and submitted to App Store.
  • iOS project under azura-frontend/ios/; same api_service.dart and backend URL.

Backend Connection (Mobile)

Single base URL in lib/api/api_service.dart: https://azura-backend-production.up.railway.app. Same endpoints as web. Some features use /api/v1/ in code; those routes may need to be added on the backend if not present.

Errors Fixed & Corrections

Backend

  1. CORS blocking requests — Skip redirect for API in custom_helper.php when URI contains /v1/ or /api/; set CORS early in index.php and V1.php.
  2. Login 400 “Email and password required” — In V1::auth_login(), fallback to php://input / parse_str() and $_POST for form-urlencoded body.
  3. CORS missing on 502 — CORS at entry and in controller; register_shutdown_function in index.php for API requests. Allow *.vercel.app and localhost.
  4. CSRF blocking API — Add v1/auth/login, v1/user/register, v1/create_user, v1/create_admin to csrf_exclude_uris in config.php.
  5. Redirect to frontend on API call — In custom_helper.php, only redirect when URI does not contain /v1/ or /api/.

Frontend (Web)

  1. Vercel “flutter: command not found” — In vercel.json, build command: export PATH="$PWD/_flutter/bin:$PATH" && flutter pub get && flutter build web --no-wasm-dry-run.
  2. XmlParserException — In api_service.dart: validate Content-Type before parsing; try/catch around json.decode; user-friendly errors for login, categories, products, getBanners, getProfile.
  3. Missing getBanners() — Implemented in api_service.dart; returns empty list on error.
  4. Service worker / source map / WebGL — Non-critical; no change for deployment.

Mobile

  • Backend connection — Confirmed _baseUrl points to Railway; main flows use correct /v1/ endpoints.
  • Android permissions — AndroidManifest.xml includes INTERNET and ACCESS_NETWORK_STATE.

Build & Deploy Commands

Backend (Railway):

cd azura-backend
git add . && git commit -m "message" && git push origin main

Frontend web:

cd azura-frontend
flutter pub get
flutter build web --no-wasm-dry-run
# Deploy build/web via Vercel (e.g. npx vercel --prod from build/web)

Android APK:

cd azura-frontend
flutter build apk --release
# Output: build/app/outputs/flutter-apk/app-release.apk

Store Submissions

  • Google Play Store: Android release build (APK or AAB) prepared and uploaded.
  • App Store: iOS build prepared and uploaded via App Store Connect.

Changing the Backend URL

Edit azura-frontend/lib/api/api_service.dart, constant _baseUrl = 'https://azura-backend-production.up.railway.app'. Then rebuild web and/or mobile and redeploy or re-submit to stores as needed.
